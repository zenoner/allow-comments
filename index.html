HTML

HTML can create the text content, markup, and references to other documents, such as images & videos, link to another web page.

HTML markup consists of an opening tag and closing tags. The entire piece of code is called an element. Attributes are listed after the elements, but before the corresponding value. Images and links are self-closing and are considered void tags, because they don't contain content.

More information about HTML

HTML Whitespace can turn everything into one line. Using br tag can seperate the content into multiple lines. Using p tag is another option but it is great for paragraphy.

Inline and Block elements are used to seperate content. An example of an inline element would be br tag because it just adds a break in the HTML line and wraps remaining text to the next line. An example of a block element would be p tag because it creates an invisible box around the content. Block elements also have a height and width.

Selectors

Selectors is where allows you to target specific HTML elements and apply style to them. This is helpful to use CSS to avoid unnecessary repetition that waste time. There are different types of Selectors:

  1. ID Selector
    Most powerful type of selector, beat out other types of selectors and styles. Having lower specificity selectors are easier to override.
  2. Class Selectors
    Most useful and versatile. Well supported in all browsers. Add multiple classes separated by space on HTML elements. Also JavaScript stuff you can do specifically for manipulating classes.
  3. Tag Selectors
    Most useful when changing properties that are unique to HTML element. Like setting the list style on a "ul" or tab-size on a "pre". Typically more useful to have class define styling that you can use on any HTML element.
  4. Attribute Selectors
    More useful than classes because they have the same specificity value, but they can be any attribute, not just "class". They also can have a value you can select by.
  5. Positional Selectors
    Several positional selectors beyond: nth-child; using simple expressions (like 3n="every third") you can select elements based on their position in the HTML.
  6. Pseudo Selectors
    ":empty" is one of many pseudo selectors, which you can recognize by the ":". They typically represent something you couldn't know just by the element and attributes alone. *These are different than pseudo elements, which are recognizable by "::"-they are responsible for adding things to the page by the things they select.

Object Oriented Programming

The Object Oriented Programming (OOP) is a programming that focused on using objects and approached to problem solving where all computations that are carried out using objects. It can operate the data structures or attributes that makes OOP as a valuable tool for thinking about large programming projects. The reason is because it has a way to handle the rapidly increasing size and complexity of software system and to make it easier to modify these large and complex system over time.

Here is an example. Think of a project management application. You would have a status object, cost object, and client object. Those objects would work together to provide the output that you want for your project management application to have.

Appropriate Use of Vocabulary
1. Object
A. It is a fundamental building blocks of application from OOP.
B. It is a self-contained component that contain properties and methods that needed to make a certain type of data useful. The methods are what it can do and operate while objects are what it knows.
C. For example, the object would be a person. A person has a name. That would be considered a property of the person. A person is able to do something, such as walking or driving. This would be considered a method of the person.
2. Class
A. This is a blueprint or set of instruction to build a specific type of object. Every object is build from a class. Each class should be designed and programmed to accomplish one.
B. For example: Think about classes, like baking a cake. A class is like a recipe for chocolate cake. The recipe itself is not a cake. It you do what the recipe tell you to do then you have an edible cake. That edible cake is an instance of the chocolate cake class.
3. Instance
A. A specific realization of any object.
B. It is different between object and instance. An object is a generic thing while an instance is a single object that has been created in memory.
4. Method
A. A procedure associated with a class.
B. This defines the behavior of the objects that are created from the class.
C. This is an action that object is able to operate.
D. For example, an object of the type person, created using the person class. Method associated with this class could consist of things like walking and driving. Methods are sometimes confused with functions, but there are distinct.
5. Module
A. A component of a larger system that interacts with the rest of the system in a simple. The idea is that module can be “plugged into” a system.
6. Library
A. A collection of implementations of behavior, written in terms of a language, that has a well-defined interface by which the behavior is invoked
B. For example, I studied OOP because it has library called Python Library Standard. It is alike document where you can find a solution for problem. It was good to me because I could learn how the computer programmer works.
7. Variable
A. It is the usually way to reference the stored value.
B. It can represent numeric value, characters, strings, or memory data.
C. Variable play an important role in computer programming because they enable programmers to write flexible programs. Rather than entering data directly into a program, a programmer can use variable to represent the data.
8. Inheritance
A. It is used to indicate that one class will get most of all of its features from a parent class.
B. When an object or classes is based on another object or class, using the same simple object to maintain the same behavior.
9. Method overriding
A. It is a very important part of OOP since it is feature that makes inheritance exploit its full power. Through method overriding a class may “copy” another class, avoiding duplicated code, and at the same time enhance or customize part of it. Method overriding is thus strict part of the inheritance mechanism.
B. Method overriding is a language feature that allows a subclass or child class to provide a specific implementation of a method that is already provided by one of its susperclasses or parent classes
